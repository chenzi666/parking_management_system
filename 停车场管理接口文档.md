# 停车场管理接口文档

## 1. 接口概述

本文档描述了停车场管理系统中停车场管理模块的API接口，包括停车场的增删改查功能。这些接口属于管理端接口，需要管理员权限才能访问。

## 2. 接口列表

### 2.1 获取停车场列表

#### 接口说明
获取系统中所有停车场的列表，支持分页查询。

#### 请求地址
`GET /api/v1/admin/parking-lot`

#### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| pageNum | Integer | 否 | 页码，默认为1 |
| pageSize | Integer | 否 | 每页数量，默认为10 |

#### 返回结果示例
```json
{
  "code": 200,
  "message": "查询成功",
  "data": {
    "list": [
      {
        "id": 1,
        "name": "中央广场停车场",
        "address": "北京市朝阳区中央广场地下一层",
        "totalSpaces": 500,
        "availableSpaces": 450,
        "contactPhone": "010-12345678",
        "status": 1,
        "statusText": "开放",
        "createdAt": "2023-01-01 10:00:00"
      }
    ],
    "total": 1,
    "page": 1,
    "pageSize": 10
  }
}
```

### 2.2 新增停车场

#### 接口说明
添加一个新的停车场到系统中。

#### 请求地址
`POST /api/v1/admin/parking-lot`

#### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| name | String | 是 | 停车场名称 |
| address | String | 是 | 停车场地址 |
| totalSpaces | Integer | 是 | 总车位数 |
| contactPhone | String | 是 | 联系电话 |
| latitude | Double | 是 | 纬度 |
| longitude | Double | 是 | 经度 |
| description | String | 否 | 描述信息 |
| status | Integer | 是 | 状态（0-未开放 1-开放 2-暂停营业） |

#### 请求示例
```json
{
  "name": "王府井停车场",
  "address": "北京市东城区王府井大街1号",
  "totalSpaces": 300,
  "contactPhone": "010-87654321",
  "latitude": 39.9143,
  "longitude": 116.4038,
  "description": "市中心大型停车场",
  "status": 1
}
```

#### 返回结果示例
```json
{
  "code": 200,
  "message": "添加成功",
  "data": {
    "id": 2,
    "name": "王府井停车场",
    "totalSpaces": 300,
    "availableSpaces": 300,
    "status": 1
  }
}
```

### 2.3 更新停车场信息

#### 接口说明
更新指定停车场的信息。

#### 请求地址
`PUT /api/v1/admin/parking-lot/{id}`

#### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 停车场ID（路径参数） |
| contactPhone | String | 否 | 联系电话 |
| description | String | 否 | 描述信息 |
| status | Integer | 否 | 状态 |

#### 请求示例
```json
{
  "contactPhone": "010-11112222",
  "description": "24小时营业的大型地下停车场，提供充电桩服务",
  "status": 1
}
```

#### 返回结果示例
```json
{
  "code": 200,
  "message": "修改成功",
  "data": {
    "id": 1,
    "contactPhone": "010-11112222",
    "description": "24小时营业的大型地下停车场，提供充电桩服务"
  }
}
```

### 2.4 删除停车场

#### 接口说明
删除指定的停车场。

#### 请求地址
`DELETE /api/v1/admin/parking-lot/{id}`

#### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 停车场ID（路径参数） |

#### 返回结果示例
```json
{
  "code": 200,
  "message": "删除成功",
  "data": null
}
```

## 3. 区域管理接口

### 3.1 获取停车场区域列表

#### 接口说明
获取指定停车场的所有区域列表。

#### 请求地址
`GET /api/v1/admin/parking-lot/{parkingLotId}/area`

#### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| parkingLotId | Long | 是 | 停车场ID（路径参数） |

#### 返回结果示例
```json
{
  "code": 200,
  "message": "查询成功",
  "data": [
    {
      "id": 1,
      "name": "A区",
      "description": "靠近商场入口"
    },
    {
      "id": 2,
      "name": "B区",
      "description": "靠近写字楼"
    }
  ]
}
```

### 3.2 新增区域

#### 接口说明
为指定停车场添加一个新的区域。

#### 请求地址
`POST /api/v1/admin/parking-lot/{parkingLotId}/area`

#### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| parkingLotId | Long | 是 | 停车场ID（路径参数） |
| name | String | 是 | 区域名称 |
| description | String | 否 | 区域描述 |

#### 请求示例
```json
{
  "name": "C区",
  "description": "靠近出口"
}
```

#### 返回结果示例
```json
{
  "code": 200,
  "message": "区域添加成功",
  "data": {
    "id": 3,
    "name": "C区",
    "description": "靠近出口"
  }
}
```

### 3.3 更新区域信息

#### 接口说明
更新指定区域的信息。

#### 请求地址
`PUT /api/v1/admin/area/{id}`

#### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 区域ID（路径参数） |
| name | String | 否 | 区域名称 |
| description | String | 否 | 区域描述 |

#### 请求示例
```json
{
  "name": "C区（快速通道）",
  "description": "靠近出口，快速通道"
}
```

#### 返回结果示例
```json
{
  "code": 200,
  "message": "区域更新成功",
  "data": {
    "id": 3,
    "name": "C区（快速通道）",
    "description": "靠近出口，快速通道"
  }
}
```

### 3.4 删除区域

#### 接口说明
删除指定的区域。

#### 请求地址
`DELETE /api/v1/admin/area/{id}`

#### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | Long | 是 | 区域ID（路径参数） |

#### 返回结果示例
```json
{
  "code": 200,
  "message": "区域删除成功",
  "data": null
}
```

## 4. 错误码说明

| 错误码 | 说明 |
|--------|------|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未认证/认证过期 |
| 403 | 没有权限 |
| 404 | 资源不存在 |
| 500 | 服务器内部错误 |

## 5. 注意事项

1. 所有管理端接口都需要管理员权限，请求时需要在Header中携带有效的JWT Token。
2. 删除停车场前需要确保该停车场下没有关联的车位、收费规则等数据。
3. 删除区域前需要确保该区域内没有车位。
